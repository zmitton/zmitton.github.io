Enumerators

Some of the most useful functions in ruby are enumerators. They allow the programmer to loop through an array, hash, or numerical set, while and indicate a block of to perform on it. There are many of these enumerator functions, one of the easiest to understand is the .map{|item| block } function.

map and map!

The map function performs the block once for each element in the group calling it. It creates a new array of all the returned values. If the "!" is indicated, the original array itself can be modified.

Examples:

array1 = [1,2,3,4,5]
array2 = array1.map{|x|
     x*10
}
puts array1 # [1, 2, 3, 4, 5]
puts array2 # [10, 20, 30, 40, 50]

In the above code the map function multiplies each element of array1 by 10 and outputs them as one new array which we assign to "array2". As you can see from the output array1 remains unchanged

If the exclamation point is used the original array can itself be modified instead:

array1.map!{|x| # notice the exclamation point this time
     x*10
}
puts array1 # [10, 20, 30, 40, 50]

This time the original array "array1" is modified, so there is usually no need for a second array to store the results

Although the results of this looping method may seem trivial I believe that's sort of the point. Doing this without using .map{} would take a few extra lines of code and would require a few temporary variables. Similar structures are made over and over by programers. Because of their simplicity, they tend to come up a lot. Having a built in method saves time for the coder, and the reader, while making the code smaller, slightly more efficient, and more elegant.
