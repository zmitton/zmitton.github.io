Looping In Ruby

Loops have always been one of the most important ideas that a computer can do. Loops, along with Sequences, and decisions make up the 3 structures that can create all programming logic. Having learned C++ first, the "while loop" was always the simplest. It's the easiest to understand: The block repeats while a condition is true. usually inside the loop a counter variable is incremented eventually stopping the code. Once you begin solving actual problems, it quickly becomes apparent that this "counter" variable come up over and over, which is when you learn the "for loop" basically the same as the while loop, except you define a temporary counter variable, and how to increment it concisely at the top. This makes the code faster and more readable.

With Ruby there seems to be a much different approach not just to looping, but to the language as a whole. There is an attitude toward simplifying and condensing code. This direction is emphasized by the number of different looping methods. Some of the ones that I have already been taught as a beginner are:

  1. while
  2. for
  3. each
  4. each_with_index
  5. map
  6. collect
  7. inject
  8. sort_by
  9. delete_if

This list can go on forever, and the documentation I have seen seems to have at least 30-40 methods like this.

The truth is, I'm not sure how I feel about this yet, because, although they can make your code more concise, they also cause the programmer to have to know what they do when they read them. It's nice to focus more on logic than syntax all of the time, but I'm already starting to see the over benefits of the more code.
